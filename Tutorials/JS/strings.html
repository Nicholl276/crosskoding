JavaScript Strings

Previous Next

Strings are used to store textual data.

Strings are defined by enclosing text within quotes.


Using Quotes

A JavaScript string consists of zero or more characters that are enclosed in quotes.

Example

let text = "John Doe";
Try it Yourself

You may use either single or double quotes.

Example

let carName1 = "Volvo XC60";  // Double quotes
let carName2 = 'Volvo XC60';  // Single quotes
Try it Yourself

Quotes Within Quotes

You can include quotes inside a string as long as they differ from the quotes that enclose the string.

Example

let answer1 = "It's alright";
let answer2 = "He is called 'Johnny'";
let answer3 = 'He is called "Johnny"';
Try it Yourself

Template Strings

Template strings, introduced in ES6 (JavaScript 2016), are enclosed in backticks (`This is a template string`).

They allow the inclusion of both single and double quotes without conflict.

Example

let text = `He's often called "Johnny"`;
Try it Yourself

String Length

To determine how many characters a string contains, use the built-in length property.

Example

let text = "ABCDEFGHIJKLMNOPQRSTUVWXYZ";
let length = text.length;
Try it Yourself

Escape Characters

Because strings are defined by quotes, JavaScript might misinterpret certain characters.

Example

let text = "We are the so-called "Vikings" from the north.";
Try it Yourself

For instance, without proper escaping, a string could be cut off as "We are the so-called ".

To address this, use a backslash (\) as an escape character, which converts special characters into literal characters.

Code Result Description
\' ' Single quote
\" " Double quote
\\ \ Backslash

Example

\" inserts a double quote in a string:

let text = "We are the so-called \"Vikings\" from the north.";

\' inserts a single quote in a string:

let text= 'It\'s alright.';

\\ inserts a backslash in a string:

let text = "The character \\ is called backslash.";
Try it Yourself

There are six additional escape sequences that JavaScript supports.

Code Result
\b Backspace
\f Form Feed
\n New Line
\r Carriage Return
\t Horizontal Tabulator
\v Vertical Tabulator

Breaking Long Lines

For enhanced readability, developers typically avoid writing excessively long lines of code.

A common practice is to break a statement safely after an operator.

Example

document.getElementById("demo").innerHTML =
"Hello Dolly!";
Try it Yourself

A safe way to break up a string is by using string addition:

Example

document.getElementById("demo").innerHTML = "Hello " +
"Dolly!";
Try it Yourself

Template Strings Revisited

As mentioned, template strings were introduced in ES6 (JavaScript 2016) and are enclosed in backticks.

They also support multi-line text without the need for explicit line break characters.

Example

let text =
`The quick
brown fox
jumps over
the lazy dog`;
Try it Yourself

JavaScript Strings as Objects

Usually, JavaScript strings are primitive values created directly from literals.

Example

let x = "John";
Try it Yourself

However, strings can also be instantiated as objects using the new keyword.

Example

let y = new String("John");
Try it Yourself

Example

let x = "John";
let y = new String("John");
Try it Yourself

When using the == operator, x and y are equal:

let x = "John";
let y = new String("John");
Try it Yourself

When using the === operator, x and y are not equal:

let x = "John";
let y = new String("John");
